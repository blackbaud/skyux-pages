Page headers describe page contents and help users navigate between parent pages and child pages. See documentation for how to setup demos.

Use page headers when users need to drill into more information from a parent page. For example, users may need to access details about an individual's giving history from the contact record or a filterable list of all reccuring gifts from a Contributions landing page.

Don't use page headers when users need to stay in context or complete quick, specific tasks. For example, do not drill into a child page when a smart action suggests that users send reminders to contacts who registered for an event. Use flyouts or modals instead.

Don't use page headers to link records in different parts of the system. For eample, users on an event registration page may need to update designations for the event, but designation management is not a child of events. Use flyouts or modals or instead.
